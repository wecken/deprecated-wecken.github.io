Mercedes-Benz User Experience (MBUX)

2019

Role

At Mercedes-Benz Research and Development, I am currently responsible for UI concepts (incl. wireframes, information architecture design, and user interaction flow design.) of the brand's GPS navigation system. The product is called MBUX. See more about it on the link.

Challenges

The biggest challenge my team had was too many "Japan-specific" features. The product and development process have been too complicated due to "over-localized system".

Design Strategy

I asked ourselves "are those features really necessary today?". To find out, I conducted an online survey. I asked Japanese users how much they are still using such conventional features that are unique to Japan. As a result, I succeeded to reduce amount of over-localized features.

This resulted significant improvements of system stability even in early stage of development and testing.

Achievements

Privious model was featuring address input method where user had to choose it step by step from prefecture to street number. This feature was unique to Japanese market. However, this method took about 90 seconds to complete average Japanese address.

Therefore, I redesigned the interaction flow so user can immediately start typing the destination using software keyboard.

With the new concept, it took just 10 seconds to complete the same address making it 9 times faster than previous model.

Because the keyboard allows voice input as well, user can enter the address even safer and faster.

After research including internal / external survey, I concluded that 10 key Japanese input method is more common and easier to use for average Mercedes-Benz customers compared to traditional 50 key layout.

So, I redesigned software keybard based on what's already common with smartphones.

Eijiro (英辞郎) on the Web Pro

2016

Role

In Castalia Co., Ltd. I was responsible for Project Management, and UI/UX Design for OEM development. This project was to redesign the iOS/Android dictionary app called "Eijiro (英辞郎) on the Web", which is one of the most popular web dictionary in Japan.

Challenges

Before I redesigned the dictionary app, there were too many screen transitions. Because the app is famous for the amount of example sentenses, I wanted the app to be more efficient when user is searching for a best suitable result from many of thouse.

Design Strategy

I wanted to have fluent animation to realize efficient interaction between master and detail views. Because of animated transition, user won't feel getting lost as a result of the interaction.

Achievements

The app is getting high user review on the App Store and Google Play store despite the fact that it's been more than 3 years since it was released.

Goocus - Mobile Learning Platform

2015

Role

Another project I worked on at Castalia was called Goocus.

I was fully responsible for its UI design, including web sites.

Challenges

Even though there are so many online courses, and a lot of them are free, it's not so easy to motivate ourselves to keep learning. We set this problem as what we solve.

Design Strategy

To solve our challenges, I included a lot of "gamification" techniques such as ranking, commenting, personalized notifications, sound effectsetc.

Achievements

Goocus is used as a platform for distributing online courses by various companies, and getting great feedback from them.
