Mercedes-Benz User Experience (MBUX)

Until end of 2019

Role

At Mercedes-Benz Research and Development, I was responsible for UI concepts (such as wireframes, and information architecture) of the brand's GPS navigation system. The product is called MBUX. See more about it on the link.

Challenges

When I joinded the company, the product was catastropic. Even in the production release, the entire system constantly kept crashing, which, as a matter of course, resulted a number of customer complaints.

First, I had to find out why that situation was happening.

The biggest problem I found was too many "Japan-specific" features. Due to historical reason, Japanese GPS navigation system evoloved in peculiar way. Therefore, the product and the process of development have been so much complicated. I had to align the Japan-specifi UI to global UI

Design Strategy

I asked myself "are those Japan-specific features really necessary?". To find out, I conducted an online survey. I asked Japanese users how often they user such conventional features. As a result, I found a lot of those features were not used anymore. Therefore, I reduced a number of those over-localized features.

This resulted significant improvements of system stability, and it allowed me to import many advanced features that were standard in other countries.

Achievements

Previous generation was featuring address input method where user had to choose destination address step by step from state to street number. This feature was unique to Japanese market. However, this method was ineficient. It took about 90 seconds to input average Japanese address.

Therefore, I redesigned the interaction flow so user can immediately start typing the destination, which accepts many types of keywords like address, POI name, POI category, POI phone number, and ZIP code, using software keyboard.

With the new concept I designed, it took just 10 seconds to complete the same address, making it 9 times faster than previous generation.

Because the keyboard allows voice input as well, user can input the address even safer and faster while driving.

After research including internal / external survey, I concluded that '10 key flick Japanese input method' is a 'must have' feature.

Therefore, I redesigned software keyboard so it matches today's user expectation.

Eijiro (英辞郎) on the Web Pro

2016

Role

In Castalia Co., Ltd. I was responsible for Project Management, and UI/UX Design for OEM development. This project was to redesign the iOS/Android dictionary app called "Eijiro (英辞郎) on the Web", which is one of the most popular web dictionary in Japan.

Challenges

Before I redesigned it, there were too many screen transitions. Because the app is famous for the amount of example sentenses, I wanted the app to be more efficient when user is searching for a best suitable result from many of those.

Design Strategy

I wanted to have fluent animation to realize efficient interaction between master and detail views. Because of animated transition, user won't feel getting lost as a result of the interaction.

Achievements

The app is getting high user review on the App Store and Google Play store despite the fact that it's been more than 3 years since it was released.

Goocus - Mobile Learning Platform

2015

Role

Another project I worked on at Castalia was called Goocus.

I was fully responsible for its UI design, including web sites.

Challenges

Even though there are so many online courses, and a lot of them are free, it's not so easy to motivate ourselves to keep learning. We set this problem as what we solve.

Design Strategy

To solve our challenges, I included a lot of "gamification" techniques such as ranking, commenting, personalized notifications, sound effects, etc.

Achievements

Goocus is used as a platform for distributing online courses by various companies, and getting great feedback from them.
